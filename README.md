## Project Structure
```bash
- **.expo, .qodo, .vscode**: Configuration and metadata for Expo, Qodo, and VSCode.
- **android, ios**: Native project folders for Android and iOS platforms (generated by Expo prebuild).
- **app**: Main application source code, including screens and navigation (see below for more details).
- **assets**: Static assets such as images, fonts, and icons.
- **components**: Reusable React components used throughout the app.
- **config**: Configuration files and settings for the project.
- **constants**: Application-wide constants and static values.
- **data**: Static or mock data used in the app.
- **db**: Database-related files or local storage logic.
- **hooks**: Custom React hooks for shared logic.
- **lib**: Utility libraries and helper functions.
- **node_modules**: Installed npm dependencies (auto-generated).
- **scripts**: Custom scripts for automation or project setup.
- **utils**: Utility functions and helpers.
- **.env, .env.example**: Environment variable files (example and actual).
- **.eslintrc.json, .prettierrc**: Linting and formatting configuration.
- **.gitignore**: Specifies files and folders to be ignored by git.
- **.nvmrc**: Node version manager configuration.
- **app.config.ts, app.json**: Expo and app configuration files.
- **babel.config.js**: Babel compiler configuration.
- **common.d.ts, expo-env.d.ts**: TypeScript declaration files.
- **eas.json**: Expo Application Services (EAS) build configuration.
- **package.json, package-lock.json**: Project dependencies and scripts.
- **README.md**: Project documentation.
- **tsconfig.json**: TypeScript configuration.

**app directory details:**
- Contains the main navigation and screen structure.
- Example subfolders: `(tabs)` for tab navigation, `preferences` for user settings, etc.
- Each folder typically contains screens (e.g., `profilescreen.tsx`, `roleselection.tsx`) and layout files (`_layout.tsx`).

```

## Development Builds

### Local Development Build
```bash
# Install the dev client
npx expo install expo-dev-client

# Generate native folders (Android/iOS)
npx expo prebuild

# Create a development build locally
npx expo run:android # For Android
npx expo run:ios     # For iOS (requires macOS)
```

### EAS Development Build
```bash
# Install EAS CLI (if not already done)
npm install -g eas-cli

# Login to your Expo account
eas login

# Configure your project (if not already done)
eas build:configure

# Create a development build
eas build --profile development --platform android # For Android
eas build --profile development --platform ios     # For iOS

# After build completes, install on device or simulator
eas build:run -p android # For Android
eas build:run -p ios     # For iOS
```

### EAS Preview Build (Staging)
```bash
# Create a preview (staging) build for internal testing
eas build --profile preview --platform android # For Android
eas build --profile preview --platform ios     # For iOS
```

### EAS Production Build
```bash
# Create a production build
eas build --profile production --platform android # For Android
eas build --profile production --platform ios     # For iOS

# Optional: Submit to app stores
eas submit --platform android # For Android
eas submit --platform ios     # For iOS
```

## Project Reset
When you're ready to start fresh, run:
```bash
npm run reset-project
```
This command will move the starter code to the **app-example** directory and create a blank **app** directory for your development.
